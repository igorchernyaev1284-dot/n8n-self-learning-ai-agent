# Самообучающийся ИИ-консультант поддержки с RAG и эскалацией на эксперта (n8n + Pinecone)

Архитектурное решение для автоматизации клиентской поддержки и ответов на типовые вопросы в Telegram-сообществах. Система снижает нагрузку на команду поддержки (экспертов) за счет автоматических ответов ИИ на основе базы знаний, при этом полностью исключая риск генерации ложных ответов (галлюцинаций) благодаря гибридному подходу **Human-in-the-Loop**.

---

## Бизнес-ценность решения

1. **Экономия ФОТ и времени экспертов:** Рутинные и повторяющиеся вопросы закрываются ИИ автоматически. Эксперты привлекаются только к уникальным или сложным кейсам.
2. **Защита от галлюцинаций ИИ:** В отличие от стандартных чат-ботов, данный агент жестко ограничен базой знаний. Если точного ответа в базе нет, система не придумывает его, а бесшовно перенаправляет запрос человеку.
3. **Автоматическое наполнение базы знаний (Self-Learning):** База знаний наполняется естественным путем в процессе работы. Эксперту достаточно один раз ответить пользователю в Telegram — система сама векторизует этот ответ и запишет в векторную базу данных для последующего использования.

---

## Архитектура и логика работы

Сценарий реализован на платформе автоматизации **n8n** и состоит из двух ключевых контуров:

### 1. Контур обработки входящих вопросов (RAG & Фильтрация)
* **Фильтрация спама и флуда:** Входящие сообщения из Telegram-группы проходят через легковесный JS-фильтр, который отсекает ботов, слишком короткие сообщения и реплики, не являющиеся вопросами.
* **Семантический поиск (Pinecone):** Одобренный вопрос векторизуется через OpenAI Embeddings и сопоставляется с базой знаний в Pinecone.
* **Интеллектуальная маршрутизация (AI Agent):** ИИ-агент (на базе `gpt-4o-mini`) анализирует результаты поиска. 
  * Если найден релевантный ответ с высоким порогом уверенности (score > 0.75), бот автоматически отправляет его пользователю в группу.
  * Если уверенность низкая или совпадений нет, бот отправляет пользователю вежливое уведомление («Уточняю у эксперта...») и инициирует процесс эскалации.

### 2. Контур эскалации и самообучения (Human-in-the-Loop)
* **Создание тикета:** Система генерирует уникальный ID тикета, сохраняет контекст вопроса во внутреннем хранилище n8n (Static Data) и отправляет интерактивную карточку с вопросом в приватный чат экспертов.
* **Перехват ответа:** Эксперт отвечает на карточку обычным реплаем прямо внутри Telegram.
* **Авто-векторизация:** n8n перехватывает этот ответ, связывает его с исходным вопросом из хранилища, формирует структурированный документ с метаданными (дата, ID тикета, язык), векторизует его и сохраняет в Pinecone.
* **Доставка ответа:** Одновременно с этим ответ эксперта автоматически отправляется пользователю в исходную группу в качестве реплая на его вопрос.

## Технологический стек

* **Оркестрация сценариев:** n8n (Self-Hosted / Cloud)
* **Векторная база данных:** Pinecone (индекс с поддержкой неймспейсов)
* **Модели ИИ:** OpenAI API (`gpt-4o-mini` для агента, `text-embedding-3` для эмбеддингов)
* **Интерфейс взаимодействия:** Telegram Bot API (работа с группами и приватными чатами)
* **Язык обработки данных:** JavaScript (ноды n8n Code для парсинга JSON, фильтрации текста и управления стейтом тикетов)

## Как развернуть проект

1. **Импорт сценария:**
   * Создайте новый воркфлоу в n8n.
   * Скопируйте содержимое файла `workflow.json` из этого репозитория.
   * Вставьте скопированный JSON на холст n8n (Ctrl+V / Cmd+V).

2. **Настройка учетных данных (Credentials):**
   * Подключите ваш Telegram Bot API в нодах триггера и отправки сообщений.
   * Укажите API-ключ OpenAI в нодах моделей ИИ и эмбеддингов.
   * Укажите API-ключ Pinecone и имя вашего индекса в нодах векторного хранилища.

3. **Конфигурация переменных:**
   * В ноде `Build Expert Card` укажите ID вашего приватного чата с экспертами в переменной `expertChatId`.
